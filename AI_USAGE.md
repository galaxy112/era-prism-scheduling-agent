# AI_USAGE

## 使用的 AI 工具

- 豆包：用于阅读论文，理解论文的核心思想。
- Codex：用于拆解题面、设计最小可运行系统、生成 Python 代码、编写 README/HTML 报告，并协助运行测试。

## AI 分别帮助了什么

- 论文迁移：把 ERA 抽象为“候选生成、执行、评分、搜索改进”，把 PRISM 抽象为“结构化协议、模拟验证、反馈修复”。
- 系统设计：将题面要求拆成基线排程器、验证器、评分器、候选生成器、协议转换器和模拟器。
- 代码实现：生成标准库 Python 代码和单元测试。
- 文档整理：生成 README、HTML 汇报材料。

## 人工修改或否决

- 没有直接照搬论文中的复杂系统，而是人工收敛为面试可演示的最小闭环。
- 否决了真实 LLM API 调用，原因是作业重点在闭环机制，可复现性比在线生成更重要。
- 保留题面原始场景和评分公式，避免引入额外解释成本。
- 对候选算法进行了人工约束：每个候选都必须可解释，并且能清楚对应一次 ERA 式改写或搜索变化。
- 测试和示例输出经过运行验证，确保 README 中的命令可执行。
